## 任务目标
你现在只负责“先生成记忆点”，不要生成对话。

请围绕该事件在时间轴上的发生过程，先确定这次事件最值得保留的记忆，再输出 memory_points。
这些记忆点将作为后续对话生成的依据，因此必须：

1. 与事件时间强相关
- 所有记忆点都应锚定在该事件的时间范围内
- primary 记忆优先对应事件发生时的核心事实
- secondary 记忆优先对应事件结束后沉淀出的偏好、情绪、计划、态度
- interference 记忆时间要与真实记忆接近，但内容必须是“合理但错误”的版本
- interference 记忆应尽量贴近对应真实记忆的时间点，优先做“同一天附近 / 同一时间段 / 同一节日前后 / 同一周内”的轻微偏差，不要跳到很远的时间

2. 与事件内容强相关
- 记忆点必须来自该事件本身、事件参与者、事件结果、事件相关图片线索
- 不要重复 system memories
- 不要生成空泛记忆，例如“今天过得不错”“和人交流了”
- 如果可用图片不为空，优先吸收其中最相关的 1-3 张图像信息，形成“可检索、可引用”的记忆点
- 若图片是 app 截图，要把它当成用户在事件中真实看到/使用过的界面证据
- 若图片是群聊截图，可以提炼“群里讨论了什么、谁说了什么类型的信息”，但不要捏造截图里不存在的细节
- 若图片是 scenery 辅助图，只能作为环境、地点、天气、提醒类辅助线索，不要把它写成事件唯一证据

3. 为后续对话服务
- 这些记忆点应该适合在后续对话中被自然提及
- 但不要把记忆写成对话语气
- 要写成简洁、明确、可检索的记忆陈述

## 记忆点要求

请生成以下三类记忆点：

### 1. Primary 记忆（2-4条）
- `memory_source`: `"primary"`
- `memory_type`: `"Event Memory"`
- 直接描述事件中真实发生的核心事实
- `importance`: 0.6-0.9
- `timestamp`: 优先贴近事件开始或关键发生时刻
- 如果与图片强相关，可在 `image_refs` 中填写相关文件名
- 如果某张图片明显支撑了该事件的关键事实，优先在 primary 记忆中引用它

### 2. Secondary 记忆（2-3条）
- `memory_source`: `"secondary"`
- `memory_type`: `"Dialogue Memory"` 或 `"Preference Memory"`
- 描述该事件中沉淀出的情绪、偏好、态度、习惯、计划
- `importance`: 0.4-0.7
- `timestamp`: 优先贴近事件后段或结束时刻

### 3. Interference 记忆（1-3条）
- `memory_source`: `"interference"`
- `memory_type`: `"Event Memory"`
- 内容必须与真实事件足够接近，但事实不正确
- 错误应该细微、合理，不能一眼看出是假的
- `importance`: 0.3-0.6
- `timestamp`: 与对应真实记忆接近
- interference 必须与真实记忆保持主题相关，优先围绕同一件事、同一场景、同一批人物、同一类计划或同一类消费展开
- interference 的错误优先来自以下几类轻微偏差：
  - 对象轻微偏差：把相关人物、礼物对象、讨论对象、出行对象、消费对象轻微混淆
  - 顺序轻微偏差：把先后顺序、前后步骤、聊天发生顺序、准备顺序写反或轻微错位
  - 年份轻微偏差：把“今年/去年/上次/前一次”混淆，但仍保持主题高度相近
- interference 要有检索迷惑性：读起来应像是同一主题下很容易记混的版本，而不是完全无关的新内容
- 不要把 interference 写成荒谬错误，也不要写成和真实记忆差得太远的平行事件
- 不要优先制造地点大幅跳变、人物大范围替换、事件性质完全改变这类一眼可排除的错误
- 如果能通过“时间接近 + 主题接近 + 仅一处关键事实轻微偏差”构造 interference，优先使用这种形式

## 输出格式要求
- `memory_id` 按格式：`{uuid}_{event_id}_{source_prefix}_{index}`
- `source_prefix` 只能使用：`pri` / `sec` / `int`
- `timestamp` 格式：`2025-01-01 10:00:00`
- `is_update`：只有在明确更新旧信息时才为 true，否则为 false
- `original_memories`：若 `is_update=true`，填被更新的 memory_id 列表，否则为空数组

## 输出格式
请严格输出 JSON：

```json
{
  "memory_points": [
    {
      "memory_id": "...",
      "memory_source": "primary",
      "memory_type": "Event Memory",
      "memory_content": "...",
      "timestamp": "2025-01-01 10:00:00",
      "importance": 0.8,
      "is_update": false,
      "original_memories": [],
      "image_refs": []
    }
  ]
}
```

注意：
- 只输出 `memory_points`
- 不要输出对话、摘要、解释
- 所有内容使用中文
- `image_refs` 中只能填写“可用图片列表”里出现过的文件路径，不能编造新路径
