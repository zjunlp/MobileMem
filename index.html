MobileMem acts as the user's personal memory agent, automatically understanding user intent, performing cross-time facial recognition to associate the daughter's identity across years, locating the 2016 birthday scene from a vast photo archive, and returning the photo of the family of three gathered around the birthday cake.

MobileMem Features

MobileMem 的特色 MobileMem Features

01 端侧长期记忆评测 On-Device Memory Benchmark

MobileMem 面向端侧个人助手，系统评测移动记忆架构的核心能力，包括层次化记忆组织、知识图谱构建、记忆合成与巩固、持续更新以及资源感知检索。

MobileMem evaluates the core capabilities of mobile memory architectures, including hierarchical memory organization, knowledge graph construction, memory synthesis and consolidation, continual updating, and resource-aware retrieval.

02 异构多模态移动轨迹 Heterogeneous Mobile Trajectories

MobileMem 不只评测人机对话记忆，而是把移动端历史建模为异构动作流：自然语言对话、应用操作、界面状态、通知、应用记录，以及截图和视觉内容。

MobileMem models mobile history as a stream of heterogeneous actions: utterances, app operations, interface states, notifications, application-specific records, screenshots, and visual content.

03 知识驱动隐私合成 Knowledge-Grounded Synthesis

KEME 将碎片化 user-app 会话作为知识锚点，在用户画像知识和时间约束引导下，组织成长时序、一致且持续演化的交互轨迹，并生成高质量问答。

KEME treats fragmented user-app sessions as knowledge anchors, organizing them into coherent, long-horizon, evolving trajectories under persona knowledge and temporal constraints, then producing high-quality QA pairs.

04 OPPO 小布评测闭环 OPPO Assistant Evaluation Loop

MobileMem 已接入 OPPO AI 助手研发流程，用真实用户轨迹、多语言和多模态任务，持续评估端侧记忆中间件、RAG 策略和端到端助手体验。

MobileMem has been integrated into OPPO's AI assistant development pipeline to evaluate on-device memory middleware, RAG strategies, and end-to-end assistant experiences with realistic user trajectories.

Core Framework

KEME 将零散的用户-App 会话作为基础知识锚点，反映已经发生且必须保留的内容。在用户画像知识和时间约束的引导下，KEME 将这些锚定会话分层组织成统一的交互流。

KEME treats the fragmented user-app sessions as foundational knowledge anchors that reflect what has already occurred and must be preserved. Guided by user persona knowledge and temporal constraints, KEME hierarchically organizes these anchored sessions into a unified interaction stream.

A:plan - Knowledge-Guided Planner

从人物画像根节点构建时序事件图，并递归展开：粗粒度生活阶段 -> 更细的子事件图 -> 叶级会话，并按拓扑顺序展开。

Builds a temporal event graph from the persona root and recursively expands it: coarse life phases →finer sub-event graphs →leaf-level sessions, expanded in topological order.

A:ground - Knowledge Anchor Grounder

将每个锚定的 App 会话分配到时间兼容的事件节点；若没有节点适配，则修订事件图，使每个锚点都可落地，并向下传播非矛盾约束。

Assigns each anchored app session to a time-compatible event node; when none fits, it revises the graph so every anchor is groundable, propagating a non-contradiction constraint downward.

A:realize - Experience Realizer

在叶节点采纳或合并锚定内容，或合成上下文一致的人机对话，并通过版本跟踪属性和证据链接驱动人物画像演化。

At leaf nodes it adopts or merges anchored content, or synthesizes context-consistent human-assistant dialogue, and drives persona evolution with version-tracked attributes and evidence links.

A:revise - Experience-Driven Reviser

每次展开后，它会更新剩余图，增添、移除或调整未来事件与边，以解决不一致并丰富结构，形成自底向上的反馈循环。

After each expansion it updates the remaining graph —adding, removing or adjusting future events and edges to resolve inconsistencies and enrich structure, forming a bottom-up feedback loop.

Persona & Memory

为平衡真实性、多样性与可扩展性，MobileMem 采用混合方式构建人物画像：一半来自代表性受雇参与者并参考真实参与者数据，另一半由这些真实样例生成虚拟画像。统一 Schema 进一步组织基础属性、上一年状态和关键里程碑，形成核心记忆结构，共 8 个真实画像和 8 个虚拟画像。

To balance authenticity, diversity, and scalability, MobileMem builds personas through a hybrid approach: half are derived from representative hired participants that reference real participant data, and half are virtual personas generated from those real examples. A unified Schema then organizes basic attributes, previous-year status, and key milestones into the core memory structure —8 real and 8 virtual personas in total.

Knowledge Graph

每个人物画像由三部分表示：基础信息向量 B 捕捉核心身份与背景，初始状态向量 S 捕捉交互前的状态和日常模式，个人知识图谱 G 则由社会关系构建。G 以人物为中心，以频繁且重要的联系人作为周围节点；每个节点描述由 GPT-5.1 丰富，并通过 Seedream 文生图模型转化为参考照片。

Each persona is represented by three components: a basic information vector B capturing core identity and background, an initial state vector S capturing status and daily patterns before interaction, and a personal knowledge graph G built from social relationships. G centers on the persona, with frequent and significant contacts as surrounding nodes; every node description is enriched by GPT-5.1 and turned into reference photos via the Seedream text-to-image model.

Event Building

在人物知识图谱驱动下，MobileMem 分三阶段生成构成每条轨迹主干的事件：重要日期收集锚定个人和文化上重要的日期；事件构建将其扩展为个性化的一年事件序列；步骤拆解把每个事件分解为按时间排序的子事件步骤，并为每一步标注涉及的移动应用和图像类型。

Driven by the persona knowledge graph, MobileMem generates the events that form the backbone of each trajectory in three stages: Important Date Collection anchors personally and culturally significant dates; Event Building expands them into a personalized one-year event sequence; and Step Breakdown decomposes every event into temporally ordered sub-event steps, each annotated with the mobile apps and image types involved.

Dialogue Synthesis

事件步骤通过两个阶段变成具体的多模态对话。记忆点拆解将每个步骤分解为细粒度文本和视觉记忆点；随后会话生成让 GPT-5.1 作为 agent，将它们编织成自然对话轮次和匹配图像，这些图像通过 HTML 渲染、文生图和图像编辑工具生成，并用参考照片保持人物一致性。会话按时间顺序拼接成完整轨迹。

Event steps become concrete multimodal dialogues in two phases. Memory Point Decomposition breaks each step into fine-grained textual and visual memory points; Session Generation then lets GPT-5.1 act as an agent to weave them into natural dialogue turns and matching images —produced via HTML rendering, text-to-image, and image-editing tools, with reference photos keeping characters consistent. Sessions are concatenated in temporal order into the full trajectory.

Question Generation

为系统评估记忆能力，评测问题基于生成的记忆点和步骤信息构建，覆盖七类：单跳、多跳、知识更新、时间推理、隐式偏好、拒答和视觉推理。

To systematically assess memory systems, evaluation questions are built from the generated memory points and step information. They span seven categories: Single-Hop, Multi-Hop, Knowledge Update, Temporal Reasoning, Implicit Preference, Abstention, and Visual Reasoning.

Application

MobileMem 的应用场景

Application Scenarios

01 生态系统 Ecosystem 现有移动智能体生态与产品管线 Existing mobile agent ecosystem and product pipelines

MobileMem 提供了一个统一的评测框架，可无缝接入现有移动智能体生态，包括任务理解、记忆检索 QA、网页搜索、回复安全和执行模块。通过与记忆中间件和持续学习框架对齐，它能够自然融入真实产品管线，并为 AI 驱动的移动体验带来快速迭代周期。

MobileMem provides a unified benchmarking framework that seamlessly integrates with the existing mobile agent ecosystem, including task understanding, memory retrieval QA, web search, response safety, and execution modules. By aligning with memory middleware and continuous learning frameworks, it naturally fits into real product pipelines and enables rapid iteration cycles for AI-powered mobile experiences.

02 标准化 Standardization 数据格式、评测协议与版本管理 Data formats, protocols, and version management

该基准已经在单一框架内统一了基础记忆、认知记忆、偏好记忆和视觉推理，同时将评测建立在真实用户数据、一年时间跨度和多样化多模态事件流之上。随着数据格式、评测协议、问题类型边界和版本管理的持续标准化，MobileMem 为发展成为行业级标准乃至国家标准奠定了坚实基础。

The benchmark already unifies foundational memory, cognitive memory, preference memory, and visual reasoning within a single framework, while grounding evaluation in real user data, one-year temporal spans, and diverse multimodal event streams. With continued efforts to standardize data formats, evaluation protocols, question-type boundaries, and version management, MobileMem lays a solid foundation for evolving into an industry-wide standard and ultimately a national standard.

03 端侧智能 On-Device Intelligence 真实约束下可度量的助手体验 Measurable assistant experience under real-world constraints

通过将长期记忆评测从能力展示转向可度量、可比较、可迭代的评估，MobileMem 直接支撑移动厂商定义端侧智能并评估个人助手体验。它为记忆质量、检索准确率和推理鲁棒性提供清晰指标，这些指标对于在真实约束下调优和部署端侧模型至关重要。

By shifting long-term memory evaluation from capability demonstration to measurable, comparable, and iterative assessment, MobileMem directly informs how mobile manufacturers define on-device intelligence and evaluate personal assistant experiences. It provides clear metrics for memory quality, retrieval accuracy, and reasoning robustness, which are essential for tuning and deploying on-device models under real-world constraints.

04 用户记忆价值 User Memory Value 以长期多模态记忆作为个性化核心 Long-term multimodal memory as personalization core

从本质上看，该基准将用户的长期多模态记忆作为个性化的核心单元。通过对真实用户轨迹、多语言交互和长时间跨度真实生活事件进行建模，它使 AI 智能体能够提供真正有帮助、具备上下文感知并持续演化的辅助能力。这将移动 AI 的价值主张从通用回复转向深度个性化、记忆驱动的体验，同时尊重用户隐私并适应个人生活方式。

At its core, the benchmark prioritizes the user's long-term multimodal memory as the central unit of personalization. By modeling authentic user trajectories, multilingual interactions, and real-life events over extended periods, it enables AI agents to deliver genuinely helpful, context-aware, and evolving assistance. This shifts the value proposition of mobile AI from generic responses to deeply personalized, memory-driven experiences that respect user privacy and adapt to individual lifestyles.

05 OPPO 应用场景 OPPO Application Scenarios OPPO 生态中的生产级移动场景 Production-grade mobile scenarios in OPPO's ecosystem

MobileMem 已经接入 OPPO 的 AI 助手研发流程，用于系统性评估并增强生产级移动场景中的长期记忆能力。在 OPPO 生态内，该基准支撑多个实际用例。第一，它使 OPPO 能够使用真实用户轨迹和个性化多模态对话，在多跳推理、知识更新、时间推理和视觉推理等多样真实任务中评估端侧记忆中间件性能。第二，该基准通过提供清晰、可复现且反映真实用户交互模式的指标，支持 OPPO 对检索增强生成（RAG）策略和记忆管理策略进行迭代优化。第三，MobileMem 覆盖英文和中文的双语评测框架与 OPPO 的全球市场战略相一致，确保记忆系统在不同语言和文化语境下保持一致表现。第四，该基准促进与 OPPO 现有智能体生态的无缝集成，包括任务理解模块、安全过滤器和执行引擎，从而实现对记忆增强型 AI 助手的端到端验证。通过将评测建立在旅行规划、社交互动、购物活动和内容消费等真实用户行为之上，MobileMem 帮助 OPPO 提供更具上下文感知、更个性化、更注重隐私的 AI 体验，使其能够在长时间跨度内真正适应用户不断演化的生活方式和偏好。

MobileMem has been integrated into OPPO's AI assistant development pipeline to systematically evaluate and enhance long-term memory capabilities in production-grade mobile scenarios. Within OPPO's ecosystem, the benchmark serves multiple practical use cases. First, it enables OPPO to assess on-device memory middleware performance across diverse real-world tasks, including multi-hop reasoning, knowledge updating, temporal reasoning, and visual reasoning, using authentic user trajectories and personalized multimodal dialogues. Second, the benchmark supports OPPO's iterative optimization of retrieval-augmented generation (RAG) strategies and memory management policies by providing clear, reproducible metrics that reflect actual user interaction patterns. Third, MobileMem's bilingual evaluation framework, which covers English and Chinese, aligns with OPPO's global market strategy, ensuring that memory systems perform consistently across different linguistic and cultural contexts. Fourth, the benchmark facilitates seamless integration with OPPO's existing agent ecosystem, including task understanding modules, safety filters, and execution engines, enabling end-to-end validation of memory-augmented AI assistants. By grounding evaluation in real-world user behaviors, such as travel planning, social interactions, shopping activities, and content consumption, MobileMem helps OPPO deliver more context-aware, personalized, and privacy-conscious AI experiences that genuinely adapt to users' evolving lifestyles and preferences over extended periods.
